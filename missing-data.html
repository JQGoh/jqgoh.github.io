Dealing with Missing Data

Why do we need to convert missing records to NaN (Not a Number)?

April 23, 2016

A newly collected data set is always messy such that it has missing records, numeric data entries marked with symbols indicating null values. Pandas package is handy in this case such that it can automatically convert the missing values (blank record) to "Not A Number" - NaN. Users can still focus on the other important features without omitting a particular data entry just because one of its features has a missing record.

For example, let us refer to the weather.csv data available for the Kaggle competition: “Predict West Nile virus in mosquitos across the city of Chicago”. The weather data describes weather conditions which may affect the presence of West Nile virus. If we refer to the summary of this data set, we only have 6 features marked as numeric data as shown below.

If we have a closer inspection of the raw data, we notice that the feature "Depart" has a lot of missing entries marked as M instead. The non-missing records of this feature may still be useful and we would like to retain them for the analysis purpose. By default, if we import the .csv file into a Pandas DataFrame, feature having both text (because of 'M' - missing) and numeric data is treated as an object data type, and it will be processed as text data.

Pandas's to_numeric() method becomes handy in this case that we can easily convert the entries of numbers to numeric data type, and mark those entries of the same feature as NaN  instead. One can easily combine the dropna() method of DataFrame to remove any row in which a null value is present. For this purpose, I designed a function: check() to utilize all the features with numeric format and remove the rows with the presence of NaN values. This will effectively reduce the number of data entries from 2944 to 1294 counts. There is a risk of discarding data entries having a limited number of NaN values, and this is not optimum for the analysis purpose. Nonetheless, we can understand the proportion of our data which has complete records without missing data. The example code of this post is available on my GitHub page as well.
